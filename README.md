# Инструкция — Статический репозиторий для учёта книг и дневника чтения

Структура репозитория:
```
/index.html            - список прочитанных книг (читает data/books/index.json)
/diary.html            - дневник чтения (читает data/diary/index.json)
/assets/style.css      - стили
/data/books/           - markdown-файлы книг и index.json
/data/diary/           - markdown-файлы записей дневника и index.json
/data/images/          - обложки и фото
/README.md             - эта инструкция
```

Как добавлять новые записи (вы будете добавлять через GitHub):
1. Добавить markdown-файл для книги в `data/books/`, например `my-book-1.md`.
   Пример содержимого (см. файл `data/books/sample-book.md` в репо).
2. Положить обложку в `data/images/` (например `my-book-1.jpg`).
3. Добавить запись в `data/books/index.json` в формате:
```json
{
  "id": "my-book-1",
  "title": "Название книги",
  "author": "Автор",
  "file": "my-book-1.md",
  "cover": "data/images/my-book-1.jpg",
  "preview": "Короткая заметка/цитата"
}
```
Порядок массива в `index.json` определяет порядок показа.

Для дневника:
1. Добавить markdown-файл в `data/diary/` (напр. `2025-11-22-01.md`).
2. Добавить запись в `data/diary/index.json`:
```json
{
  "date": "2025-11-22",
  "title": "Наброски по главе 3",
  "book": "Название книги (если привязано)",
  "file": "2025-11-22-01.md"
}
```

Замечания:
- Сайт — статический. Чтобы он корректно работал при просмотре напрямую из репозитория, лучше развернуть через GitHub Pages (ветка `gh-pages` или `main`).
- Если хочется не редактировать `index.json` вручную, можно добавлять небольшие `index`-утилиты, но сейчас всё максимально просто — вы добавляете файлы и обновляете индекс.
- В `data/books/` и `data/diary/` уже есть примеры.

Если хотите — могу добавить простой скрипт на GitHub Actions, который будет автоматически обновлять `index.json` по содержимому папок (тогда не надо править index.json вручную). Напишите — и я добавлю.
